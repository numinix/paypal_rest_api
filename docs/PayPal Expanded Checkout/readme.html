ChatGPT Agentic Product Feed for Zen Cart

Connect your store to ChatGPT—where your data becomes insights, suggestions, and real customer answers.


Overview

The ChatGPT Agentic Product Feed for Zen Cart generates curated product data designed for OpenAI-powered agents and assistants. The module exposes an intuitive admin experience that mirrors the look and feel of other Numinix dashboards while giving you granular control over which catalog items are discoverable by conversational AI.

Use this document to install the plugin, configure the catalog rules, and automate feed generation so ChatGPT always has access to your latest inventory.

  • Validated with Zen Cart 1.5.5e through 1.5.8, with the dashboard UX optimized for 1.5.6 and newer releases.
  • Requires PHP 7.2.34+ with cURL and JSON extensions enabled; syntax checks pass through PHP 8.x.
  • Outputs a JSON feed purpose-built for ChatGPT Action integrations.

Compatibility & requirements

The module adheres to Zen Cart APIs introduced prior to 1.5.5e and avoids modern PHP features such as arrow functions, typed properties, union types, and nullsafe operators so stores on long-term support stacks can deploy without core upgrades.

For production use we recommend PHP 7.3 or newer to take advantage of upstream security fixes, but stores running PHP 7.2.34 remain supported. When deployed on PHP 8.x environments the exporter continues to build feeds successfully, making it suitable for hosts that have already upgraded.

Installation

Preparing your store

  1. Install the Numinix Premium Plugin Installer to add the nxPluginLicCheck dependency required by the module bootstrap.
  2. Backup your Zen Cart database and catalog files before making any changes.
  3. Verify that your hosting environment meets the PHP requirements listed in the Overview.
  4. Ensure your admin directory name is updated in the package’s YOUR_ADMIN/ folder before uploading.

Uploading files

  1. Unzip the distribution package locally.
  2. Upload the includes/ and YOUR_ADMIN/ contents to the matching directories in your store, merging with existing files.
  3. Place chatgpt_product_feeder_run.php in the root of your Zen Cart catalog.

Database updates

  1. Log in to your Zen Cart admin.
  2. Navigate to Tools → Install SQL Patches.
  3. Upload and execute uninstall.sql only if you are replacing an older beta build. Fresh installs do not require manual SQL patches.

After the files are in place, the admin menu will display Tools → ChatGPT Agentic Product Feed and the configuration group will appear under Configuration.

Configuration

Open Configuration → ChatGPT Agentic Product Feed to tailor how the module maps your catalog to the ChatGPT schema.

  1. Store identity

    • Populate your storefront name, support email, and primary storefront URL.
    • Select the default currency and language that the feed should reference.

  2. Catalog scope

    • Choose which categories are eligible for exposure to ChatGPT.
    • Enable or disable inclusion of out-of-stock products.
    • Define default shipping and fulfillment notes that help the agent answer logistical questions.

  3. Product enrichment

    • Map custom fields or attributes to the standardized ChatGPT product properties (brand, condition, identifiers, etc.).
    • Optionally enable the AI Highlights composer to inject short persuasive blurbs into the feed.

  4. Security

    • Generate a signed URL token to restrict access to the feed endpoint.
    • Restrict the admin tool to specific Zen Cart administrator profiles if desired.
Usage

Once configured, head to Tools → ChatGPT Agentic Product Feed to work with the interactive dashboard.

  1. Review the overview tiles

    At the top of the screen you’ll find totals for included products, pending refreshes, and the timestamp of the most recent feed build.

  2. Filter the catalog

    Use quick filter chips or open the advanced drawer to combine rules for category, stock status, pricing, keywords, and refresh state.

  3. Curate inclusion

    Toggle products in or out of the ChatGPT feed using the row-level switch or the bulk toolbar actions. Mark specific items for refresh to regenerate their payload during the next run.

  4. Generate the feed

    Click Build Feed to produce the latest JSON file. The admin will surface the public URL and signed token (if enabled) so you can register it with your ChatGPT Action or agent manifest.

Virtual product support

Catalog items flagged as virtual automatically omit weight, dimensional, and shipping attributes. The exporter marks them as in_stock with a minimum quantity of 1 so digital goods publish without requiring physical fulfillment details.

Feed registration with OpenAI

  1. Create or edit your ChatGPT Action in the OpenAI dashboard.
  2. Set the data source URL to the feed endpoint provided by the module.
  3. Specify the refresh schedule that matches your automation cadence.
  4. Test the Action to ensure product responses match your storefront content.
Automation

Automate regeneration so your ChatGPT Action stays synchronized with inventory updates.

  1. Edit your server’s cron table (crontab -e).
  2. Add a schedule that matches how frequently your catalog changes. For example:
0 * * * * /usr/bin/php /path/to/zen-cart/chatgpt_product_feeder_run.php

The runner script executes the same logic as the admin build button, respecting inclusion choices and refresh flags. Review your server logs periodically to confirm the job executes without errors.

Troubleshooting
  • Empty feed? Confirm that at least one category is enabled and that the inclusion toggle is switched on for the products you expect.
  • Encoding issues? Ensure product descriptions and custom fields use UTF-8 and remove any invalid characters before saving.
  • Timeouts when building? Reduce the batch size in configuration or increase max_execution_time in php.ini.
  • Permission errors? The module writes feed files to /feed/ by default. Confirm the directory is writable by the web server user (typically 0755 or 0775).
  • Security reminders: Protect the feed directory with HTTP authentication or IP allowlists if you expose the URL publicly. Rotate signed tokens whenever staff access changes.
