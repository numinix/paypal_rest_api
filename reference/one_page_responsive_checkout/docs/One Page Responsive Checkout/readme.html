One Page Responsive Checkout

by Numinix


Installation
  1. For upgrading versions prior to 3.5.5 of ONE PAGE RESPONSIVE CHECKOUT, you need to install first Numinix Premium Plugin Installer for Zencart
  2. Backup your database and all files.
  3. Install Numinix Premium Plugin Installer for Zencart
  4. Install CSS JS Loader v6.0.0+
  5. Upload all files in this package while maintaining the directory structure (rename YOUR_TEMPLATE to your custom template folder name)
  6. Login to your admin and the configuration menu should be automatically created under CONFIGURATION > ONE PAGE RESPONSIVE CHECKOUT CONFIGURATION.
  7. If using a SEO URLs module, add "checkout_payment", "one_page_checkout", and "one_page_confirmation" to the exempt list. In other words, do not rewrite the URLs for these pages.
  8. Go to Admin -> Configuration -> , Sessions, and set Force Cookie Use to True.
  9. Go to Admin -> Configuration -> Customer Details, and set "State field - Display as pulldown when possible" to True.
  10. The plugin now automatically disables the left and right side columns for the one-page checkout and confirmation pages; no template edits are required.
  11. If you need to restrict additional custom pages while using the guest checkout feature, add the following to that page's header_php.php file (for example, includes/modules/pages/gv_redeem/header_php.php):
    Select code
    $zco_notifier->notify('NOTIFY_HEADER_REGISTERED_USERS_ONLY'); // OPRC
    Note: the following pages are already restricted by the plugin: account, account_edit, account_history, account_history_info, account_notification, account_password, address_book, address_book_process, gv_send, gv_redeem

  19. To use GOOGLE reCAPTCHA (v2) with the OPRC registration form:
    1. Create a site in the Google reCAPTCHA Admin Console and choose the "I'm not a robot" Checkbox option for reCAPTCHA v2.
    2. Copy the Site key and Secret key generated for your domain.
    3. In your Zen Cart admin area, go to Configuration > One Page Responsive Checkout and enter the keys in the reCAPTCHA Site key and reCAPTCHA Secret key settings.
    4. Enable Enable reCAPTCHA Form and select the desired theme.

    The checkout template automatically loads the required https://www.google.com/recaptcha/api.js script once reCAPTCHA is enabled, so no additional template edits are necessary.

Upgrade
  • Upgrading from v3.6.7 and earlier
    The core edits documented above for includes/classes/order.php, includes/classes/payment.php, and the admin order management pages remain unchanged. However, starting in v3.7.0 the following adjustments are different from prior releases:
    • The plugin now disables the side columns on checkout and confirmation automatically. Remove any manual edits previously made to includes/templates/YOUR_TEMPLATE/common/tpl_main_page.php.
      • Delete the conditional logic that hides column_left.php or column_right.php when $current_page_base equals one_page_checkout or one_page_confirmation.
      • Restore the stock require(DIR_WS_MODULES . zen_get_module_directory('column_left.php')); and column_right.php statements (copy them from your unmodified template if necessary).
    • Shipping quote caching is handled internally. Restore the core includes/classes/shipping.php file if you previously merged plugin changes.
      • Remove any // OPRC blocks that load custom cache helpers (for example, require(DIR_WS_CLASSES . 'OPRC_shipping_cache.php');).
      • Replace the customized file with the unmodified version from your Zen Cart distribution so the class matches the core implementation.
    • Update to CSS JS Loader v6.0.0 or newer to ensure compatibility with the bundled jQuery resources.
  • 3.7.1 — Side columns for the one-page checkout and confirmation pages are now disabled automatically, removing the need for manual template edits. Guest checkout sessions now clean up automatically after checkout success.
Configuration
  1. Confirm the module is enabled under Configuration > One Page Responsive Checkout Configuration, then review each option to align with your store policies. Recommended defaults include enabling Guest Checkout, Address Book Synchronization, and Persistent Cart so shoppers can finish quickly without losing their selections.
  2. Upload a transparent PNG or SVG of your brand (up to roughly 240px wide by 90px tall) to a secure location, then enter that address into Checkout Logo Secure URL. The checkout will load the image through Zen Cart’s zen_image helper so optimization plugins can resize or cache it while the stylesheet keeps the logo responsive without distorting its aspect ratio.
  3. Define the checkout layout you want customers to see by setting Default Checkout Tab and enabling only the panels you actively use (e.g., coupons or gift certificates). Disabling unused panels keeps the interface focused and prevents confusion.
  4. Review the payment and shipping method options to ensure they expose any required surcharges or descriptions. Pair each method with clear language and, where possible, enable the built-in Shipping Method Info fields outlined above to improve conversion.
  5. If you rely on third-party tracking or analytics, enable the Order Summary Tracking Hooks and verify the callback URLs before going live. This ensures scripts execute once and only after a successful checkout.
  6. After saving configuration changes, clear template cache files and test the checkout flow in an incognito/private window to verify that both guest and registered customer experiences follow the expected path.
Tips
  • Document your chosen values from Configuration > One Page Responsive Checkout Configuration before deploying to production so you can quickly reapply them after updates.
  • Create overrides for template adjustments — for example, copy custom CSS into includes/templates/YOUR_TEMPLATE/css/auto_loaders/one_page_checkout_overrides.css so upgrades never overwrite styling.
  • Test the checkout path as a guest and as a logged-in customer after enabling new payment or shipping modules. Confirm required fields (like state/province) appear and that confirmation emails render correctly.
  • When using SEO/URL rewriting plugins, exclude checkout_payment, one_page_checkout, and one_page_confirmation from rewrites to avoid stalled sessions and cached payment selections.
  • Whitelist numinix.com on your firewall or security plugins to prevent license validation calls from being blocked.
Troubleshooting
  • Checkout page does not load — Ensure the CSS JS Loader dependency is updated to v6.0.0+ and clear the /cache directory so stale scripts are not referenced.
  • Payment methods appear blank — Verify each module's JavaScript validation uses check_payment_form() as outlined above and disable conflicting one-page checkout add-ons.
  • Guest checkout loops back to login — Confirm Force Cookie Use is set to True and whitelist checkout pages in any security/redirect plugins.
  • Shipping rates show incorrect totals — Remove custom caching edits from includes/classes/shipping.php so the built-in quote refresh logic can run.
  • License validation errors — Make sure outgoing connections to numinix.com are allowed and that server time is accurate. Re-saving the license key after correcting connectivity will reinitialize the check.
Uninstall
  1. Delete all uploaded files from your store.
  2. Revert all file edits performed during the installation.
  3. Patch your database by copying and pasting the contents of uninstall.sql into ADMIN > TOOLS > INSTALL SQL PATCHES
