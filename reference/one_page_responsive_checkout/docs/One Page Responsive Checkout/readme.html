One Page Responsive Checkout


Installation
  1. Backup your database and all files.
  2. ionCube Loader must be installed. IF needed the The IonCube loader wizard zip file is included in this package, review the readme for instructions
  3. Install CSS JS Loader v2.0.0+
  4. Upload all files in this package while maintaining the directory structure (rename YOUR_TEMPLATE to your custom template folder name)
  5. If using a SEO URLs module, add "checkout_payment", "one_page_checkout", and "one_page_confirmation" to the exempt list. In other words, do not rewrite the URLs for these pages.
  6. Go to Admin -> Configuration -> Sessions, and set Force Cookie Use to True.

Installation Tips
  • Read through each of the descriptions for each setting in ZEN CART ADMIN > CONFIGURATION > ONE PAGE RESPONSIVE CHECKOUT CONFIGURATION.

    From here you can turn the module on or off, as well as enable or disable various features of the module.
  • For easier upgrading, never modify the stylesheets included with this module (i.e. one_page_checkout.css). Instead, create a file named one_page_checkout_overrides.css and upload to /includes/templates/YOUR_TEMPLATE/css/auto_loaders/.
  • If Guest Checkout is enabled, the option will only be available if there is an item in the customers shopping cart, otherwise it will default to Register Customer.
  • If using Ultimate SEO URLs, make sure that the option "Enable automatic redirects?" is set to "false".
  • Disable your SEO URLs mod for the "one_page_checkout" page (refer to your SEO URLs mod's documentation for instructions on how to disable for specific pages).
  • To setup a Google Analytics Funnel Visualization, first enable Google Analytics in the configuration and select the method your Google Analytics integration is using. Then build two funnels. The first for New Customers:
    • Name: Login or Create Account, Screen/Page: /oprc_hideregistration_register
    • Name: Create Account, Screen/Page: /oprc_create_account
    • Name: Checkout, Screen/Page: /index.php?main_page=one_page_checkout
    • Name: Confirmation, Screen/Page: /one_page_confirmation (if you are not using one page checkout, then you should change this value to /index.php?main_page=one_page_confirmation)
    The second for Returning Customers:
    • Name: Login, Screen/Page: /oprc_login
    • Name: Checkout, Screen/Page: /index.php?main_page=one_page_checkout
    • Name: Confirmation, Screen/Page: /one_page_confirmation (if you are not using one page checkout, then you should change this value to /index.php?main_page=one_page_confirmation)
  • Note to white list numinix.com for license
Uninstall
  1. Remove edits performed in the installation procedure.
  2. Delete all uploaded files from your store.
  3. Patch your database by copying and pasting the contents of uninstall.sql into ADMIN > TOOLS > INSTALL SQL PATCHES
