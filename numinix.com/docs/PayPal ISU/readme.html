PayPal Integrated Sign-up (ISU)

Embedded PayPal onboarding for Zen Cart by Numinix


Installation


Introduction

PayPal Integrated Sign-up (ISU) delivers a modern, white-labeled onboarding flow that connects Zen Cart merchants to PayPal Complete Payments without leaving the admin area. The module communicates directly with the PayPal Partner APIs, relays onboarding status to the storefront, and directs merchants to retrieve credentials directly from PayPal once approval is complete.

The ISU experience aligns with PayPal brand standards, enforces HTTPS-only callbacks, and guides merchants from initial eligibility checks through go-live readiness. Built on native Zen Cart patterns, it avoids encrypted loaders and keeps all logic transparent for compliance reviews.

This document explains how to deploy the onboarding tools, configure environment-specific settings, monitor events, and support merchants as they complete the PayPal activation journey.

Requirements

Before launching PayPal onboarding from your Zen Cart admin, confirm the following prerequisites:

  • Zen Cart 1.5.7 or newer with the admin auto-installer enabled so configuration groups and menu entries can be registered.
  • PHP 7.4+ or PHP 8.x with curl, openssl, and json extensions active to support secure API calls.
  • HTTPS enabled for the admin area; ISU refuses to run over insecure HTTP when NUMINIX_PPCP_FORCE_SSL is set to true (recommended).
  • Access to PayPal partner credentials (sandbox and live client IDs and secrets) that are issued to your PayPal partner account.
  • Outbound connectivity from the server to PayPal API domains (for example https://api-m.sandbox.paypal.com and https://api-m.paypal.com) so onboarding sessions can be created and polled.
Installation
  1. Back up your storefront files and database, paying special attention to the admin configuration tables.
  2. Upload the module's files to your Zen Cart installation, preserving the directory structure under includes/, management/, and docs/.
  3. Log into the Zen Cart admin to trigger the auto-installer. A success message confirms that the Numinix PayPal Integrated Sign-up configuration group and admin tools were created.
  4. Navigate to Admin > Configuration > Numinix PayPal Integrated Sign-up and review each setting before merchants begin onboarding.
  5. Populate the sandbox and live partner credentials supplied by PayPal. Save the values directly in the configuration panel so signup links and status checks authenticate correctly.
  6. Launch the onboarding wizard from Modules > Payment > PayPal (Complete Payments) or the Tools > PayPal Signup Link Generator page to validate connectivity in sandbox before inviting merchants.
Upgrade from manual PayPal onboarding flows

Stores migrating from email-based credential exchanges or custom partner flows can adopt ISU with minimal downtime:

  1. Back up your current PayPal credentials and remove any hard-coded keys from templates or configuration overrides.
  2. Deploy the latest ISU files and log into the admin so database migrations 1.0.0 through 1.0.2 can run.
  3. Confirm that the configuration group displays the environment selector, SSL enforcement toggle, and partner credential fields with accurate values.
  4. Update your merchant-facing instructions to direct them to the embedded onboarding button inside the PayPal Restful payment module configuration.

When these steps are complete, you can deprecate any legacy partner forms or manual credential storage processes in favour of the streamlined ISU workflow.

Configuration Overview


The installer registers a configuration group inside Admin > Configuration > Numinix PayPal Integrated Sign-up. Review these keys whenever the PayPal partner program updates requirements:

  • Environment selection: Choose sandbox or live per request using the toggle on the PayPal signup form or by passing the env parameter.
  • Numinix PPCP Force SSL: Enforces HTTPS-only onboarding requests. Leave enabled unless debugging in a secure staging tunnel.
  • Partner Client IDs and Secrets: Supply sandbox and live pairs issued by PayPal so the onboarding widget can authenticate against the correct environment.
  • Numinix PayPal ISU Version: Tracks the installed schema version and should not be edited manually.

Future upgrades adjust descriptions and add new settings automatically while retaining partner credential values.




Onboarding session lifecycle

The paypal_signup page coordinates each onboarding session and automatically manages state:

  • Merchants launch the flow by clicking the onboarding button. The module requests a tracking ID from PayPal via the partner-referrals API and stores it in the session.
  • The wizard polls /status until PayPal signals completion or cancellation. Admins can review progress via AJAX responses.
  • When onboarding finishes successfully, PayPal confirms completion through the merchant-integrations API and distributes API credentials directly to the merchant account owner via the PayPal dashboard.
  • Telemetry pings record errors and key events through NOTIFY_NUMINIX_PAYPAL_ISU_EVENT, enabling observers to audit or extend the workflow.

If a merchant closes the window prematurely, they can relaunch the flow and the module will resume with the existing tracking ID when available.




Operational Notes

Credential handling

Beginning with v1.0.5 the storefront no longer persists API credentials. PayPal issues secrets directly to the merchant after approval, so no credential payloads are stored in Zen Cart.

API connectivity

Firewalls must permit outbound HTTPS traffic to PayPal partner endpoints such as https://api-m.sandbox.paypal.com and https://api-m.paypal.com. Failures surface in the onboarding modal and are also written to the PayPal ISU log file at logs/numinix_paypal_signup.log.

Extensibility

Listen for NOTIFY_NUMINIX_PAYPAL_ISU_EVENT notifications to add custom analytics, send webhooks, or integrate third-party CRMs when merchants reach specific milestones.

Uninstall


  1. Review the NUMINIX_PPCP_<ENV>_* partner credential entries and clear them if you no longer require access before removing files.
  2. Delete the configuration group entries from Admin > Configuration > Numinix PayPal Integrated Sign-up or run the provided uninstall SQL to drop related keys.
  3. Remove the admin menu entries (configNuminixPayPalIsu, customerNuminixPaypalIsuSaveCreds, toolsNuminixPaypalIsuSignupLink) via the Admin Access Management page if they are no longer needed.
  4. Delete the module files under includes/modules/pages/paypal_signup/, includes/classes/observers/, and management/.
Release Notes


Version 1.0.6 – Direct PayPal API onboarding

This release removes the dependency on the hosted proxy and moves storefront flows to PayPal's Partner APIs.

  • Issues partner referrals directly against PayPal and surfaces the referral identifier to the storefront session.
  • Polls merchant integration status via PayPal endpoints to determine waiting, cancelled, or completed steps.
  • Refreshes deployment, troubleshooting, and FAQ guidance to emphasise PayPal connectivity.

Version 1.0.5 – Hosted proxy alignment

This release simplifies the ISU footprint for numinix.com by removing storefront credential storage.

  • Streamlines storefront storage by relying on PayPal for credential distribution.
  • Retires plugin/standalone mode selection and the credential persistence service.
  • Updates storefront UI, telemetry, and documentation to direct merchants to PayPal for credential retrieval.

Version 1.0.2 – Operational hardening

This maintenance release focuses on reliability ahead of expanded merchant onboarding.

  • Normalises admin menu entries across Zen Cart versions and honours the optional menu_display column.
  • Updates credential configuration labels and descriptions to clarify sandbox versus live usage.
  • Ensures the plugin mode and SSL enforcement defaults remain in place when upgrading from earlier builds.

Version 1.0.1 – Admin tooling updates

This release polishes the initial ISU roll-out for partner-managed stores.

  • Adds dedicated admin menu items for the credential synchronisation and signup link utilities.
  • Introduces the Numinix PayPal Integrated Sign-up configuration group with environment, SSL, and mode settings.
  • Stores partner credential placeholders and a version key to simplify future migrations.

Version 1.0.0 – Initial release

The inaugural PayPal ISU package embeds the partner onboarding journey inside Zen Cart.

  • Delivers the paypal_signup controller with AJAX orchestration for start, status, and finalisation steps.
  • Persists PayPal credentials via the credential persistence service when plugin mode is enabled.
  • Provides admin tools for regenerating signup links and confirming stored credentials.

Migrating from legacy onboarding

When transitioning from manual partner enrollment or legacy PayPal modules:

  1. Audit existing PayPal credentials and determine whether they should be rotated after the ISU cut-over.
  2. Disable any storefront links that pointed merchants to external forms and replace them with the embedded onboarding button.
  3. Verify that the PayPal Restful payment module exposes the integrated signup button for merchants.

These steps align your merchant support team with the new PayPal-branded onboarding journey and reduce time-to-live for new accounts.




Troubleshooting

The onboarding modal does not launch

Verify outbound access to the PayPal API endpoints from the server and confirm that your browser allows pop-ups for the admin domain. Errors encountered before launch are recorded in logs/numinix_paypal_signup.log.

Merchants finished onboarding but still need credentials

Beginning with v1.0.5 the storefront no longer persists API credentials. Ask the merchant to log into PayPal > Apps & Credentials to retrieve the keys issued during onboarding, or contact PayPal Partner Support if the account is still pending review.

Status polling never completes

Long-running waiting states typically mean outbound requests to PayPal were blocked. Confirm port 443 access for the web server and review PayPal Partner dashboard telemetry for the recorded tracking ID.

How do I temporarily disable onboarding?

Hide the onboarding call-to-action in your storefront templates or restrict access to the signup page via admin tools. Existing merchants remain active while the entry point is disabled.

FAQ

How can I restart onboarding for a merchant?

  1. Open the PayPal Restful payment module in the Zen Cart admin and click Launch PayPal onboarding.
  2. If the merchant previously started onboarding, confirm whether a tracking ID appears in logs/numinix_paypal_signup.log.
  3. Use the PayPal Signup Link Generator tool to email a fresh link if the merchant is working outside the admin.

Where are credentials and logs stored?

API credentials are delivered directly by PayPal after onboarding—this module no longer stores them. Operational logs continue to live inside logs/numinix_paypal_signup.log. Purge sensitive data according to your organisation's retention policy.

Can I point ISU at a custom onboarding endpoint?

No. The onboarding flow now connects directly to PayPal's Partner APIs and does not support alternate proxies or self-hosted endpoints.
